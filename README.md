### 医療機関リストの管理

#### 設定画面を開く

1. 画面右上の **「+ リスト追加」** ボタンをクリック
2. 「詳細設定」パネルが開く
3. 「医療機関リスト管理（スプレッドシート連携）」セクションで操作

![詳細設定パネル](docs/images/P1_settings-panel.png)

---

#### 医療機関を追加する

1. 「新しい医療機関名を入力...」欄に名前を入力
2. **「+ 追加」** ボタンをクリック（またはEnterキー）

![医療機関追加](docs/images/add-hospital.png)

3. リストに追加され、スプレッドシートとGoogleフォームも自動更新

![医療機関リスト](docs/images/hospital-list.png)

> 追加ボタンを押した際、入力欄が空の場合は入力欄にフォーカスが移動します。

---

#### 医療機関名を編集する

1. 編集したい医療機関にマウスを合わせる
2. 右側に表示される鉛筆アイコンをクリック
3. 名前を編集
4. 保存アイコンをクリック、またはEnterキーを押す（Escキーでキャンセル）

---

#### 医療機関を削除する

1. 削除したい医療機関にマウスを合わせる
2. 右側に表示されるゴミ箱アイコンをクリック
3. 確認ダイアログで「OK」をクリック

> **注意**: 削除するとスプレッドシートとGoogleフォームの選択肢からも削除されます

---

#### リストを更新する

他の人が追加・編集した内容を反映するには：

1. 「医療機関リスト管理」の横にある「更新」をクリック
2. 最新のリストが読み込まれる

> リストは画面を開いた時にも自動で取得されます。

### URLのコピー・共有

生成されたフォームURLを共有したい場合：

1. 医療機関を選択
2. QRコード確認エリアに表示されるURLの横にある **「コピー」** ボタンをクリック
3. URLがクリップボードにコピーされる
4. メールやチャットに貼り付けて共有

> URLには医療機関名がprefill（事前入力）パラメータとして埋め込まれています。

## 4. アンケートの質問構成

### 紙アンケート（1ページ目）

| 質問番号 | 内容 | 回答形式 |
|----------|------|----------|
| 質問1 | 名前（カタカナ） | 氏・名の記述式 |
| 質問2 | 生年月日 | 昭和・平成・令和 + 年月日の記述式 |
| - | 以降、QRコードで回答 | チェックボックス（質問2の行の右端に配置） |
| 質問3 | 性別 | 男 / 女 / 回答しない |
| 質問4 | 血液型 | A型 / B型 / O型 / AB型 / わからない |
| 質問5 | 身長・体重 | 記述式（cm / kg） |
| 質問6 | 糖尿病 | なし / 5年未満 / 5〜10年前 / 10年以上前 / わからない |
| 質問7 | 脂質異常症 | なし / 5年未満 / 5〜10年前 / 10年以上前 / わからない |
| 質問8 | 兄弟に糖尿病歴はありますか？ | はい / いいえ / わからない |
| 質問9 | 両親に糖尿病歴はありますか？ | はい / いいえ / わからない |
| 質問10 | ほとんど運動しない | はい / いいえ |
| 質問11 | お菓子、スイーツなどを週何回食べますか | ほぼ毎日 / 週2〜3回 / 週1回以下または食べない |
| 質問12 | 最もよく飲む飲み物は何ですか？ | 有糖飲料 / 無糖飲料 |
| 質問13 | 飲酒習慣 | 飲む（種類・頻度・量を記入）/ ほとんど飲まない |
| 質問14 | 【医師入力欄】歯の抜去位置 | 右/左・上/下 + 記述式 |

#### 「以降、QRコードで回答」チェックボックスについて

質問2（生年月日）の行の右端に「□ 以降、QRコードで回答」というチェックボックスが配置されています。これは、紙のアンケートで名前と生年月日（質問1・2）までを記入した後、残りの質問（質問3以降）はアンケート用紙のQRコードをスマートフォンで読み取ってオンライン（Googleフォーム）で回答する運用を想定したものです。

- 患者さんがこのチェックボックスにチェックを入れた場合、質問3以降は紙に記入せず、QRコードからオンラインで回答します
- チェックを入れない場合は、引き続き紙のアンケートで全質問に回答します
- この運用により、紙とオンラインを併用した柔軟なアンケート実施が可能になります

### 飲酒習慣 選択肢一覧（2ページ目）

質問13で「飲む」と回答した場合の参考資料として印刷されます。

| 列 | 内容 | 選択肢 |
|----|------|--------|
| お酒の種類 | 9種類 | ビール、日本酒、焼酎、チューハイ、ワイン、ウイスキー、ブランデー、梅酒、泡盛 |
| 週に何回 | 7段階 | 1回〜7回（毎日） |
| サイズ/飲み方 | 11種類 | 350ml缶、500ml缶、750mlビン、375mlビン、コップ、水割り、お湯割り、ロック、小/中/大ジョッキ |
| 数量 | 6段階 | 1〜6以上 |

### API通信方式

ReactアプリとGoogle Apps Scriptの間は **JSONP形式** で通信しています。

```
[Reactアプリ]
    │
    │  <script>タグを動的に生成してGASのURLを呼び出す
    │  （CORS制限を回避するためfetch/XHRは使用しない）
    ↓
[Google Apps Script]
    │
    │  doGet(e) でリクエストを受け取り
    │  callback関数名でラップしたJavaScriptを返す
    ↓
[Reactアプリ]
    グローバルに登録したコールバック関数でレスポンスを受け取る
```

| API アクション | 説明 |
|----------------|------|
| `getHospitalList` | 医療機関リストを取得 |
| `addHospital` | 医療機関を追加（フォーム選択肢も同期） |
| `updateHospital` | 医療機関名を変更（フォーム選択肢も同期） |
| `deleteHospital` | 医療機関を削除（フォーム選択肢も同期） |

> タイムアウトは30秒に設定されています。接続に失敗した場合は「サーバーに接続できません」のエラーメッセージが表示されます。

### JSON自動保存機能

Googleフォームで回答が送信されると、Apps Scriptのトリガーにより自動的にJSON形式でGoogleドライブに保存されます。

```
[ユーザーがフォームを回答・送信]
    │
    ↓
[スプレッドシートに回答が保存]
    │
    ↓
[onFormSubmit トリガーが発火]
    │
    ├─ 回答データをJSONオブジェクトに変換
    ├─ Googleドライブの「アンケートJSON」フォルダに保存
    └─ スプレッドシートの「JSON作成済み」列に「済」フラグを記入
```

- ファイル名: `survey_{ID番号}.json`（ID番号がない場合は `survey_row{行番号}_{タイムスタンプ}.json`）
- 同じID番号のファイルが既に存在する場合は、古いファイルをゴミ箱に移動して上書き
- `processExistingDataWithoutJson()` 関数で、未処理の既存データを一括処理可能

### ファイルの保存場所

このシステムのファイルは **2つの場所** に分かれて保存されています。

```
┌─────────────────────────────┐     ┌─────────────────────────────┐
│   GitHub / Vercel           │     │   Googleドライブ             │
│   (ソースコード)             │     │   (データ・フォーム)          │
├─────────────────────────────┤     ├─────────────────────────────┤
│ ・SurveyEditor.jsx          │     │ ・Googleフォーム             │
│ ・App.jsx                   │     │ ・Google Apps Script        │
│ ・main.jsx                  │     │ ・スプレッドシート            │
│ ・Code.gs (バックアップ)     │     │   (医療機関リスト)           │
│ ・README.md                 │     │ ・スプレッドシート            │
│ ・package.json              │     │   (回答データ)              │
│ ・index.html                │     │ ・アンケートJSONフォルダ      │
│ ・vite.config.js            │     │   (回答のJSON自動保存先)     │
│ ・eslint.config.js          │     │                             │
└─────────────────────────────┘     └─────────────────────────────┘
```

#### GitHub / Vercel にあるもの

| ファイル | 説明 |
|----------|------|
| src/SurveyEditor.jsx | メインのReactコンポーネント（UI・API呼び出し・印刷処理） |
| src/App.jsx | アプリのルートコンポーネント |
| src/main.jsx | ReactDOM のレンダリング処理 |
| src/index.css | グローバルCSS |
| src/App.css | アプリ用CSS（Viteデフォルト） |
| Code.gs | Apps Scriptのバックアップ |
| README.md | このドキュメント |
| index.html | HTMLテンプレート（Tailwind CSS CDN読み込み含む） |
| package.json | 依存関係の定義 |
| vite.config.js | Viteビルド設定 |
| eslint.config.js | ESLint設定 |

#### Googleドライブにあるもの

| ファイル | 説明 |
|----------|------|
| Googleフォーム | オンライン回答用フォーム |
| Google Apps Script | サーバー処理（API） **※実際に動作する本体** |
| スプレッドシート（医療機関リスト） | 医療機関の一覧データ |
| スプレッドシート（フォーム回答） | フォーム回答データ + JSON作成済みフラグ |
| アンケートJSONフォルダ | フォーム回答のJSON自動保存先 |

> **重要**: Google Apps Script（Code.gs）は2箇所にあります
> - **Googleドライブ**: 実際に動作している本体 → **変更はここで行う**
> - **GitHub**: バックアップ用のコピー → 変更後にコピーを保存

#### 自動デプロイの仕組み

GitHubにCommitするだけで、サイトが自動更新されます。

| 順番 | 何が起きる | 所要時間 |

## 7. 管理者向け情報

### Vercelへのログイン方法

VercelはGitHubアカウントと連携しているので、GitHubにログインできればVercelにもアクセスできます。

#### ステップ1: Vercelにアクセス

1. ブラウザで https://vercel.com を開く
2. 右上の **「Log In」** ボタンをクリック

#### ステップ2: GitHubでログインを選択

1. ログイン方法の選択画面が表示される
2. **「Continue with GitHub」** をクリック

#### ステップ3: GitHubの認証

1. GitHubのログイン画面が表示される
2. **GitHubのユーザー名（またはメール）** を入力
3. **GitHubのパスワード** を入力
4. **「Sign in」** をクリック

#### ステップ4: Vercelダッシュボードが開く

1. ログイン成功後、Vercelのダッシュボードが表示される
2. **「touka-survey」** プロジェクトをクリックして管理画面へ

---

### GASデプロイ時の注意

Google Apps Scriptのコードを変更した場合は、以下の手順で再デプロイが必要です。

1. GASエディタで「デプロイ」→「新しいデプロイ」を選択
2. アクセスできるユーザーを **「全員」** に設定
3. デプロイ後、新しいURLが発行された場合は `SurveyEditor.jsx` の `API_URL` を更新
4. GitHubにコミット → Vercelに自動デプロイ

> **注意**: 「デプロイを管理」から既存のデプロイを更新する場合はURLは変わりません。「新しいデプロイ」を選ぶとURLが変わるため、ReactアプリのAPI_URLも合わせて更新する必要があります。

## 8. トラブルシューティング

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「サーバーに接続できません」 | ネットワークの問題、GASの一時的な障害、タイムアウト（30秒） | ページを再読み込みして再試行。繰り返す場合はGASデプロイURLを確認 |
| 医療機関リストが空 | スプレッドシートにデータがない、または接続エラー | 「リスト追加」から医療機関を追加、またはリスト横の更新ボタンをクリック |
| QRコードが表示されない | 医療機関が未選択、またはQRコード生成APIの問題 | 医療機関を選択し直す。外部API（api.qrserver.com）の障害の可能性もあり |
| 印刷レイアウトがずれる | ブラウザの印刷設定 | 印刷ダイアログで余白を「なし」に設定、用紙サイズをA4に設定 |
| GASデプロイ後にAPIが動かない | デプロイURLの不一致 | SurveyEditor.jsxのAPI_URLが最新のデプロイURLと一致しているか確認 |
| JSONファイルが生成されない | onFormSubmitトリガーが未設定 | GASエディタで`setupFormSubmitTrigger()`を手動実行してトリガーを設定 |
